## Root causes (agent-team diagnostic, 2026-05-14)

### RC1 — Reverse-Vds Id clamp eliminates BSIM-CMG-like reverse conduction
`pycircuitsim/models/mosfet_nn.py:430` sets `f_id = f_sym if normal_dir else 0.0` and zeroes Id/gm/gmb in the reverse-Vds branch. The original goal was to enforce `Id(Vds=0)=0` and avoid wrong-direction current at untrained reverse Vds. The unintended consequence: when the inverter's load cap rings past a rail (e.g. Vout below GND after Vin rises to VDD), the NN NMOS sees `Vds < 0` and outputs Id=0 — no restoring current. Only the PMOS rail-restoring leakage (Rule 15(a) at `|Vds_nn| > VDD_train`) pulls Vout back, which is weak.

Inspected mechanism from `results/v6_3_phaseA_metrics/inverter_tran_TSMC12.csv` at the discontinuities:
- t=260 ps, Vin just reached 0.8 V: Vout_nn = −99.8 mV (ref −0.6 mV).
- t=270 ps: Vout_nn = −33.8 mV
- t=280 ps: Vout_nn = −23.1 mV
- t=290 ps: Vout_nn = −17.2 mV
- ...exponential decay to +0.15 mV by t=500 ps, then sits stable.
- Decay τ ≈ 14 ps suggests `R ≈ 14 kΩ` (`G ≈ 70 µS`). That conductance is the PMOS Rule 15(a) `g_extra` (~140 µS at 100 mV overshoot) — too weak to suppress the overshoot quickly.

Same mechanism, mirrored, at high rail: `+18.9 mV` overshoot above VDD at small Vin.

**Earlier theory (now rejected):** I initially attributed RC1 to the unconditional `gds = max(gds, g_extra)` bump at line 422 leaking into the reverse branch. Phase A patched that and produced **bit-identical** traces — proving that branch was never hit on the converged operating points. The real driver is the Id clamp, not the gds bump.

### RC2 — `inv_trip` overlay is centered on transistor Vth, not inverter Vtrip
`external_compact_models/PyCMG/pycmg/nn_generate.py:440-469` (`_inv_trip_points`) builds the overlay around `find_threshold` (peak-gm Vth, e.g. ~0.72 V for TSMC12 NMOS) instead of inverter Vtrip ≈ VDD/2.

Audit (agent, 2026-05-14):
- TSMC12 NMOS: **0 samples** at Vg < 0.48 V. Inverter Vtrip = 0.40 V → overlay misses the band entirely.
- TSMC5 NMOS: only **0.24 %** of overlay rows in ±33 mV around VDD/2 = 0.325 V. Worst trip spike (206 mV) despite densest overlay because the band the inverter switches in barely overlaps the overlay.
- The `tech_name` gate at `nn_generate.py:629-631` was widened to include `tsmc12, tsmc16` for V6.2.1, so the prior CLAUDE.md note ("gated to TSMC5/TSMC7") is stale.

### RC3 — Zero reverse-Vds coverage in training data
Main hybrid grid (`nn_generate.py:322-334`) sweeps `Vd ∈ [0, 2·VDD]` for NMOS, mirror for PMOS. **Zero samples at Vd < 0 (NMOS) or Vd > 0 (PMOS)**. Rule 15(a) is the only thing covering NR reverse-overshoot. Removes pressure on Rule 15(a) entirely once added.

## Plan

### Phase A — Inference-only `gds`-bump gate (DISCARDED, no measurable effect)

The patch (gate `result["gds"] = max(result["gds"], g_extra)` on `normal_dir`) was applied and the eval re-run. Output `results/v6_3_phaseA_metrics/inverter_*.csv` were **bit-identical** to V6.2.1 — the reverse-Vds branch with `|Vds|>VDD_train` was never hit on any converged operating point. The patch was reverted on 2026-05-14. Keep the V6.2 sign convention as is.

### Phase B — Dataset regen + retrain (proposed)

Now the only path forward: fix both the trip-region (RC2) and the reverse-conduction (RC1+RC3) at training time, since both reduce to coverage gaps in the dataset.

**Dataset changes** (`external_compact_models/PyCMG/pycmg/nn_generate.py`):
1. `_inv_trip_points` (lines 440-469): replace `Vth_signed` center with `VDD/2`; widen `vg_lo / vg_hi` to `[0.30·VDD, 0.70·VDD]`. Confirmed by audit: TSMC12/16 currently have **zero** overlay samples in the inverter switching band; TSMC5 has only 0.24%.
2. Main grid `_sample_hybrid_grid_voltages` (lines 322-334): extend Vd range to `[-0.3·VDD, +2·VDD]` for NMOS (mirror for PMOS) to seed reverse-Vds reverse-conduction physics. Currently zero reverse-Vds samples exist. The −0.3·VDD lower bound covers the inverter's measured −100 mV ringing at VDD=0.8 (TSMC12/16) with a margin of ~2×.
3. Density: target ≥5% of total rows in `|Vds| < 0.05·VDD` AND ≥5% in `Vds ∈ [-0.2·VDD, 0]` (NMOS reverse band, mirror for PMOS). The trip overlay's existing ~22k rows/bin should suffice for the trip-region fix; the reverse-Vds corridor is a new contribution and needs a comparable budget.

**Regen and retrain:**
1. Per-tech regenerate `*_nmos.npz` and `*_pmos.npz` for TSMC5/7/12/16 via `external_compact_models/PyCMG/scripts/generate_nn_data.py --device both --tech tsmc{5,7,12,16} --enable-inv-trip --n-workers 8`.
2. Retrain 8 medium DirectNet cells via `bash scripts/train_per_tech_8cells.sh` (or its 8-cell equivalent for size=medium only — confirm script scope before invocation). Checkpoint stems remain `tsmc{X}_dn_medium_{nmos,pmos}` so the parser resolver cascade still matches without code changes.
3. Re-run `python scripts/eval_v6_2_1_inverter.py` against the new V6.3 checkpoints; write metrics to `results/v6_3_metrics_report/`.

**Expected effect:**
- VTC trip-region spike: TSMC5 206 mV → ~30 mV target; TSMC12 39 mV → ~15 mV; TSMC7/16 already ≤ 56 mV, expect proportional improvement.
- Transient pull-low undershoot: 99 mV (TSMC12/16) → near reference's ~5 mV, once NMOS reverse conduction is learned and the Rule 15(a) extrapolation can hand off to in-distribution NN behavior near the rails.

**Risks:**
- Extending the main grid Vd range without raising the row budget thins out other regions. Either keep row count constant (rebalance density) or grow the dataset; my preference is to grow ~10–15% to keep existing region density intact.
- Reverse-Vds samples cost PyCMG eval time. With `--n-workers 8` should still be hours not days.
- Per-tech embedding vocab unchanged; checkpoint stems unchanged; parser resolver unchanged. No code changes outside the data-generation files.
- TSMC5 inv_trip overlay currently covers only Vg ≥ 0.259 V. The re-center to VDD/2 will help most for TSMC12/16 where the current band is entirely above the inverter region; TSMC5/7 already have some overlap, so the relative gain may be smaller.

## Verification gates (definition of done for V6.3)

For all 4 TSMC techs, on the inverter benchmark:
- VTC `MaxErr ≤ 25 mV` (≈10× improvement at TSMC5; ≈1.5–2× at TSMC7/12/16).
- VTC trip-shift `|ΔVtrip| ≤ 5 mV` (already met at V6.2.1).
- Transient post-startup `MaxErr ≤ 30 mV`.
- No NR convergence regression on the BSIM-CMG-comparable inverter runs.
- Per-target NRMSE on the test split for `id` ≤ 0.05% on `medium` (V6.2.1 already meets this).

Both consistent with excess trip-region density.

## Phase C — Reduce inv_trip overlay weight (Vbs=0 only)

`inv_trip` is 9.83 % of every V6.3 dataset (class 7). The `grid` class already
sweeps Vbs across 5 levels everywhere; the inverter operates at Vbs=0 always,
so the ±0.25·VDD Vbs levels in `_inv_trip_points` are body-bias variation
the inverter never sees and that costs the NN capacity to fit the trip-region
slope correctly.

**Patch** (`external_compact_models/PyCMG/pycmg/nn_generate.py:445-481`):
drop the `[0, +0.25·VDD, -0.25·VDD]` Vbs sweep; keep `vg_steps = 25` and
`vd_steps = 9` at `[0.30, 0.70]·VDD`. Overlay drops from 25 × 9 × 3 = 675 →
25 × 9 × 1 = 225 samples/bin (3.3 % of total rows). Single-line minimal change;
no tech-specific branching.

**Regen + retrain:** all 8 cells (uniform reduction). TSMC5 V6.3 result (57.9 mV)
may shift slightly since its overlay density also drops, but the VDD/2
recentering — which is what unlocked TSMC5's 206 → 58 mV win — is preserved.

**Expected effect:**
- TSMC5 VTC: ~58 → ~50–70 mV (similar, density change is uniform).
- TSMC7 VTC: ~143 → ~50–60 mV (revert to V6.2.1 levels; less trip-region overweight).
- TSMC12 VTC: ~63 → ~30–40 mV (revert; reverse_vds + correct overlay weight).
- TSMC16 VTC: ~51 → ~25–35 mV (revert; cleanest trip region).
- Transient pull-low: ~55 mV maintained (reverse_vds class unchanged).

**Risks:** TSMC5 was the most dependent on dense trip-region coverage; reducing
density risks losing some of the 206 → 58 mV improvement. If TSMC5 VTC regresses
past ~80 mV, fall back to a tech-specific `vbs_levels` (e.g., 3 levels for TSMC5,
1 for TSMC7/12/16).

### Why the gates remain open — gain amplification, not coverage

Three dataset revisions (V6.2.1 → V6.3 → V6.3.1) moved the trip-region VTC
error around but never below ~45 mV. The error sits at Vin ≈ Vtrip ± 25 mV
where inverter gain ≈ −15 to −30; the NN's residual Id error (~0.05 %
test-split NRMSE) is amplified ~20× into 45–80 mV of Vout error. `inv_trip`
overlay tuning cannot break this floor — the floor is set by gm/gds fidelity
at the trip operating point. A future Phase D needs a small-signal lever
(trip-weighted gm/gds-matching loss term), not more trip samples.

### Dead ends recorded

- **Phase A** (inference-only gds-bump gate): bit-identical traces, no effect. Reverted.
- **Phase B** `_inv_trip_points` wide `[0.30,0.70]·VDD` × 3-Vbs overlay: fixed
  TSMC5 + transient but regressed TSMC7/12/16 VTC via trip-region over-fit.
  Superseded by Phase C (same centering, Vbs sweep dropped).

### Task #8 status (reverse-Vds Id clamp loosening)

Not pursued. The transient win came entirely from the `reverse_vds` dataset
class; the `mosfet_nn.py:430` `f_id=0` clamp was left intact and is no longer
the binding constraint on transient accuracy. Loosening it is a possible
Phase-D lever but was not needed to ship V6.3.1.
