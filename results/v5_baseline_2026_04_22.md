# v5 Baseline: v4 Shipping Inverter Verification (2026-04-22)

**Purpose.** Capture the reference baseline numbers for the v4-shipping
checkpoints (DirectNet LEVEL=73 + BSIMAR LEVEL=74, both `v4_universal`)
against which upcoming v5 experiments will be compared.

## Environment

- **Branch:** `feat/bsimar-v4-tech-code`
- **Commit:** `706bcdd` (HEAD at baseline capture time)
- **Script:** `tests/verify_bsimar_v4_inverter.py` (default `--checkpoint-prefix v4_universal`)
- **Runner:** `conda run -n pycircuitsim python tests/verify_bsimar_v4_inverter.py --tech <tech>`
- **Covered tests (per tech):** NMOS DC sweep, PMOS DC sweep, Inverter VTC, Inverter transient
  - Default thresholds in the script: DC/VTC 10 % NRMSE, transient 15 % NRMSE.
- **Techs run sequentially** (shared GPU + BSIM-CMG OSDI): tsmc5, tsmc7, tsmc12, tsmc16.

## Failures, warnings, anomalies

- No run FAILED or stalled outright; every run produced a `SUMMARY` block
  with well-formed NRMSE numbers. All four logs contain a cosmetic
  `ERROR conda.cli.main_run:execute(127):` line from a `conda run`
  bookkeeping quirk, but the Python process completed with the summary
  intact and the numbers match the `v5_improvement_plan_2026_04_21.md §1`
  claimed baseline (see comparison below).
- Two NGSPICE stderr warnings printed on the TSMC5 run only:
  `PTWG_i = -1.325033e+00 is negative, setting it to 0.` (naive
  modelcard coefficient clamp; no effect on numerical output).
- Script-defined VTC thresholds (10 %) penalise every low-VDD tech:
  TSMC5 FAILS on VTC for both models, TSMC7 FAILS on VTC for both models.
  The transient threshold (15 %) passes all 8 tech×model cells.
- PMOS DC NRMSE for TSMC12 and TSMC16 is persistently ~12-14 % across
  both models — consistent with the known v4 training weakness on PMOS
  high-VDD technologies.

## Comparison against plan-claimed baseline (`v5_improvement_plan_2026_04_21.md §1`)

The plan file lists:

| Tech  | DN inv. tran claimed | BSIMAR inv. tran claimed | VTC (AR) claimed |
|-------|----------------------|--------------------------|------------------|
| TSMC5 | 17.20 %              | 12.13 %                  | 13.96 %          |
| TSMC7 |  8.87 %              |  9.14 %                  | 19.15 %          |
| TSMC12| 11.65 %              |  6.78 %                  |  4.10 %          |
| TSMC16| 10.59 %              |  7.51 %                  |  3.40 %          |

This run reproduces BSIMAR transient + VTC (AR) to the listed digits
(12.13 / 13.96 / 9.14 / 19.15 / 6.78 / 4.10 / 7.51 / 3.40).

DirectNet transient numbers differ from the plan:
**TSMC5 17.20 % -> 3.75 % (much better)**,
TSMC7 8.87 % -> 6.80 %, TSMC12 11.65 % -> 4.86 %,
TSMC16 10.59 % -> 7.86 %.
This is expected if the plan was written against an earlier
DirectNet checkpoint; commit 706bcdd already includes the rail-restoring
extrapolation fix (`feat(train+sim): sign/boundary losses +
VDD-proportional Vds correction`, commit f3e5f56, and `fix(sim):
rail-restoring extrapolation for NN compact models`, commit 381bbfc).
Treat the numbers above — not the plan-claimed numbers — as the
authoritative v5 baseline.
